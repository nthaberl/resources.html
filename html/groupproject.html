Discussing WanderLoom: A Travel Photo Album Project

[ Conception and Technical Aspects ]

The idea for the WanderLoom project came from my personal interests in traveling and taking photos. I wanted an easy and fun way to compile photos from trips so I could "revisit" those trips later on. Users of Instagram often do something similar with "highlight reels" on their profiles, and traditional photo storage, like Google Photos, also provides similar functionality. However, both of these options are more involved when it comes to trying to "revisit" a trip.

This project takes a simplified approach, allowing users to create their own albums, upload photos, and view them on an interactive map. The current phase of the project focuses on core functionality: creating an account, creating albums, uploading photos, extracting information from those photos, and mapping the photos onto a map. Future implementations could include social features like collaborative albums, enhanced maps, and even visual enhancements like stickers.

The domain models for this project include users, albums, photos, and an interactive map. Firebase Authentication handles the user authentication, and users are able to create albums and upload photos, both of which are stored into Firestore. The interactive map is provided via the Mapbox API. The frontend is built with React. The selection of these tools came from teammembers' prior experience with these tools. Considerations for the project included cross-browser compatibility, responsive design for desktop and mobile, and compliance with WCAG accessibility standards.

[ Project Management and Personal Contributions ]

Project management revolved around achieving a working prototype by the end of the quarter. This meant splitting the project between core features and "nice-to-have" features. Tasks were divided via volunteer basis during our weekly meetings. When there was more work to be done, extra meetings were scheduled to ensure everything was completed on time.

Due to being the "project manager", and having prior experience with system design diagrams, my major contributions thus far included creating a majority of the design diagrams and documentation. This helped solidify my understanding of the project and helped visualize the workings of our project to my teammates. Any documentation I created was reviewed and accepted by teammates.

Creating diagrams reinforced my understanding of system archiecture, and I gained more hands-on experience with React and Firebase. I gained a deeper understaning of the importance of documentation, and collaborating with my team imrpvoed my communication and planning skills. Personal Experience

In other classes I've been reflecting deeply on how I work in teams and my contribution within them. The majority of my experiences involved me doing a majority of the work, judging ideas as they are brought to the table, and feeling burnt out and frustrated by the end of it. I've also had a handful of great experiences in teams and reflected on what made them successful, which included active communication and allowing others to contribute. For this project, I wanted to focus on being more flexible and open to ideas, and maintaining active communication. I also wanted to focus on doing my part and not overextending myself, which is something I've struggled with in the past. I felt I was able to do this because my teammembers were active participants, wanted to contribute meaningfully, and wanted to see a high quality result at the end of the project. Collaborating with them was also effortless. In previous classes, groupwork either involved creating a project from scratch, or creating robust documentation for a hypothetical project to meet user needs. This is the first time I've brought both of those things together. This was challenging because the documentation is pretty involved, so it was difficult to balance writing the documentation while getting the actual project up and running. Although challenging, it was great to get the practice of creating documentation for a project that I envisioned, instead of just writing documentation for a project that was given to me.
